# Field Notes：把探空圖講給阿嬤聽——一個小模型的誠實用法

> Build Small Hackathon 開發筆記。發表到 HF blog 前把「翻車記錄」的
> 程式輸出截圖補上,效果更好。

## 為什麼選這個題目

家裡長輩看不懂氣象 App 的雷達回波圖,但完全聽得懂「今天要帶傘、棉被先別曬」。
另一邊,全球每天施放上千顆探空氣球,產出的 Skew-T 圖卻只有受過多年訓練的人
讀得懂。SkyRead 把這兩件事接起來：探空 → 對流指數 → 兩張卡片
（同行版精確數字、生活版阿嬤語言）。

## 小模型的誠實用法

第一性原理：**4B 以下的模型算不準 CAPE,所以從一開始就不讓它算。**

- 所有數值——CAPE/CIN、LCL/LFC/EL、K、LI、TT、PWAT——由 MetPy 確定性計算
- 同行版卡片是規則式模板:專業讀者要的本來就是精確數字
- 小模型只做一件事:把一句**數值正確的草稿**改寫得更口語、更親切
- 改寫結果要通過驗證（繁體中文、禁簡體字、長度、不得回音指令）,
  不合格自動重試,重試仍不過就直接用草稿

評審問「模型幻覺怎麼辦」時,答案是:它連碰數字的機會都沒有。

## 翻車記錄（最有價值的部分）

1. **MiniCPM4-0.5B 連抄都會抄壞。** 原本想讓 0.5B 改寫整份雙層判讀,
   結果它回音指令、發明「ELITICALITIES」這種詞。把任務縮小成
   「改寫一句話」後,它還是把一句繁中改出「阿电池，今天天氣真的不对」。
   同尺寸的 Qwen3-0.6B 一次就過——0.5B 做不做得到,跟訓練配方關係比參數量大。
2. **trust_remote_code 是會腐爛的。** MiniCPM4 的自訂程式碼用了
   transformers v5 移除的 `is_torch_fx_available`;降到 4.57 後,
   MiniCPM3-4B 又踩到 `DynamicCache.seen_tokens`（4.47 移除）。
   一度 pin 在 4.46.3——直到發現這個 pin 反過來把備案模型
   Qwen3 鎖在門外（qwen3 架構要 4.51+）。教訓:用 remote code 模型,
   transformers 版本是模型規格的一部分,而且這個成本會傳染給整個專案。
3. **4B 也救不了繁體中文。** 換上 MiniCPM3-4B 後,穩定天氣的句子改得不錯,
   但只要句子裡出現「出門帶傘」這組詞,它就必定滑進簡體（出门/伞/午后）,
   龍捲風個案連續六次全滅——而那正是 demo 最重要的一案。
   one-shot 範例、強化指令、取樣重試都救不回:這不是 prompt 問題,
   是訓練分布問題。最後加了一道「高頻簡體字黑名單」驗證閘門,
   並換成 Qwen3-0.6B——九次生成全過、零簡體,還快十倍。
   小了 6 倍的模型贏在訓練配方,再次驗證:參數量不是品質的代理指標。
4. **台灣探空不在 Wyoming 資料庫。** 板橋 46692、屏東 46810 都查無資料,
   只好誠實地取最近的石垣島 47918（離台北約 270 km）,UI 上照實標註。
5. **「草稿改寫」比「自由生成」可靠一個量級。** 給小模型一份正確草稿
   要求潤飾,失敗模式從「捏造數字」變成「改得不夠好」——後者 fallback
   接得住,前者接不住。另外,改寫請用取樣而非 greedy:greedy 的失敗
   是確定性的,重試等於白做;取樣才讓「驗證 + 重試」這套保險真正生效。

## 學到什麼

把 AI 的工作面積縮到最小、再用確定性系統包住它,小模型就從「不可靠的
生成器」變成「可驗證的潤飾器」。Build small 不只是參數量小——
是讓模型只做它真正擅長的那一小塊。
